# Data Model: Local Embedding Provider Support

**Feature**: `002-local-embeddings`

No new domain entities. Existing tables are reused; the migration command changes the dimension of `chunks.embedding` and inserts a new active row in `embedding_models`.

---

## Tables touched

### `embedding_models` (unchanged schema)

- Already carries `provider`, `name`, `dimensions`, `is_active` per `001_initial_schema.sql`.
- The migrate command deactivates the current active row and inserts a new active row whose `provider`/`name`/`dimensions` match the runtime config. Single-active invariant is already enforced by `idx_embedding_models_active`.
- Extended to accept `'ollama'` in the `provider` string (no constraint change — the column is free-text today).

### `chunks` (column type changed by migrate)

- `embedding` is `vector(N)`; `N` is altered by the migrate command to match the new active model's dimensions.
- Truncated as part of the migrate command. Chunks are regenerated by the enrichment worker from entity content.

### `entities` (no schema change)

- Migrate runs `UPDATE entities SET enrichment_status='pending', enrichment_attempts=0, updated_at=now() WHERE content IS NOT NULL`.

### `audit_log` (no schema change)

- Migrate writes one row on every invocation (dry-run and real) with `operation='embeddings.migrate'` and `details` carrying previous and new model metadata, affected counts, and `dry_run: true|false`.

---

## Runtime config

Env vars added / changed:

| Var | Required | Default | Notes |
|---|---|---|---|
| `EMBEDDING_PROVIDER` | no | `openai` | `openai` or `ollama` |
| `EMBEDDING_MODEL` | no | per-provider (`text-embedding-3-small` / `bge-m3`) | |
| `EMBEDDING_DIMENSIONS` | no | per-provider (`1536` / `1024`) | Must match the active `embedding_models` row at startup |
| `EMBEDDING_BASE_URL` | no | falls back to `OLLAMA_BASE_URL` when provider is `ollama` | Embedding host, independent from LLM extraction host |
| `EMBEDDING_API_KEY` | no | none | Optional bearer for embedding host |
| `OPENAI_API_KEY` | conditional | none | Required when `EMBEDDING_PROVIDER=openai` OR (`EXTRACTION_ENABLED && EXTRACTION_PROVIDER=openai`) |

`OLLAMA_BASE_URL` and `OLLAMA_API_KEY` remain dedicated to LLM extraction.

---

## Dimension flow

| Source | Role | Startup behavior if mismatched |
|---|---|---|
| Env `EMBEDDING_DIMENSIONS` | What the operator intends | — |
| `embedding_models.dimensions` (active row) | What storage actually holds | Server refuses to start; error names both values and the migrate command |
| Actual vector length from provider at call time | Runtime validation | Existing enrichment error path; entity enters `failed` state; retries per existing behavior |
